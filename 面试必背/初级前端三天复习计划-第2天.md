     - position: absolute/fixed

### 第二模块：JavaScript进阶（2.5小时）

#### ✅ 必学知识点

1. **事件循环（Event Loop）** → [JS高频面试题汇总.md](./JS高频面试题汇总.md#18-事件循环) + [事件循环面试题详解.md](./事件循环面试题详解.md)

   **执行机制：**
   - 调用栈（Call Stack）：执行同步代码
   - 宏任务队列（Macro Task）：setTimeout、setInterval、I/O
   - 微任务队列（Micro Task）：Promise.then、MutationObserver、queueMicrotask

   **执行顺序：**
   1. 执行同步代码
   2. 执行所有微任务
   3. 执行一个宏任务
   4. 执行所有微任务
   5. 重复3-4

   **典型例题：**
   ```javascript
   console.log(1);
   setTimeout(() => console.log(2), 0);
   Promise.resolve().then(() => console.log(3));
   console.log(4);
   // 输出：1 4 3 2
   ```

2. **Promise深入理解** → [JS高频面试题汇总.md](./JS高频面试题汇总.md#20-promise-asyncawait)

   **三种状态：**
   - pending（进行中）
   - fulfilled（已成功）
   - rejected（已失败）

   **状态转换：**
   - pending → fulfilled（不可逆）
   - pending → rejected（不可逆）

   **链式调用：**
   - then：处理成功和失败
   - catch：捕获错误
   - finally：无论成功失败都执行

   **静态方法：**
   ```javascript
   // Promise.all - 全部成功才成功
   Promise.all([p1, p2, p3]).then(results => {});

   // Promise.race - 第一个完成就返回
   Promise.race([p1, p2, p3]).then(result => {});

   // Promise.allSettled - 等待全部完成
   Promise.allSettled([p1, p2, p3]).then(results => {});

   // Promise.any - 第一个成功就返回
   Promise.any([p1, p2, p3]).then(result => {});
   ```

3. **async/await** → [JS高频面试题汇总.md](./JS高频面试题汇总.md#20-promise-asyncawait)

   **基本使用：**
   ```javascript
   async function fetchData() {
     try {
       const response = await fetch('/api/data');
       const data = await response.json();
       return data;
     } catch (error) {
       console.error('Error:', error);
     }
   }
   ```

   **错误处理：**
   - try/catch 捕获
   - .catch() 捕获

   **并行执行：**
   ```javascript
   // 并行执行
   const [result1, result2] = await Promise.all([
     fetch('/api/1'),
     fetch('/api/2')
   ]);

   // 串行执行
   const result1 = await fetch('/api/1');
   const result2 = await fetch('/api/2');
   ```

4. **ES6模块化** → [JS高频面试题汇总.md](./JS高频面试题汇总.md#21-es6-模块化)

   **ES Modules：**
   ```javascript
   // 导出
   export const name = 'Vue';
   export function sayHi() {}
   export default class App {}

   // 导入
   import App, { name, sayHi } from './app.js';
   import * as utils from './utils.js';
   ```

   **CommonJS：**
   ```javascript
   // 导出
   module.exports = { name, sayHi };
   exports.name = 'Vue';

   // 导入
   const { name, sayHi } = require('./app.js');
   ```

   **区别对比：**
   | 特性 | ES Modules | CommonJS |
   |------|-----------|----------|
   | 加载方式 | 静态加载（编译时） | 动态加载（运行时） |
   | 输出 | 值的引用 | 值的拷贝 |
   | this指向 | undefined | module |
   | 使用场景 | 浏览器、现代打包工具 | Node.js |

### 第三模块：HTTP网络专项（2小时）

#### ✅ 必学知识点

1. **HTTP vs HTTPS** → [HTTP面试题精简版.md](./HTTP面试题精简版.md#1-http-vs-https-)

   **主要区别：**
   - 端口：HTTP 80、HTTPS 443
   - 安全性：HTTPS有SSL/TLS加密，HTTP明文传输
   - 证书：HTTPS需要CA证书
   - 性能：HTTPS握手耗时，但HTTP/2优化了性能

   **HTTPS加密过程：**
   1. 客户端发送Client Hello（支持的加密算法）
   2. 服务器发送Server Hello + 证书
   3. 客户端验证证书
   4. 密钥交换（非对称加密）
   5. 数据传输（对称加密）

2. **GET vs POST** → [HTTP面试题精简版.md](./HTTP面试题精简版.md#2-get-vs-post-)

   | 特性 | GET | POST |
   |------|-----|------|
   | 参数位置 | URL查询字符串 | 请求体 |
   | 数据大小 | 受URL长度限制（2KB） | 理论无限制 |
   | 安全性 | 参数暴露在URL | 相对安全 |
   | 幂等性 | 幂等（多次请求结果相同） | 非幂等 |
   | 缓存 | 可缓存 | 默认不缓存 |
   | 书签 | 可收藏为书签 | 不可收藏 |
   | 使用场景 | 查询数据 | 提交数据、修改数据 |

3. **HTTP状态码** → [HTTP面试题精简版.md](./HTTP面试题精简版.md#4-http-状态码-)

   - **2xx 成功：**
     - 200 OK：请求成功
     - 201 Created：资源创建成功
     - 204 No Content：请求成功但无返回内容

   - **3xx 重定向：**
     - 301 Moved Permanently：永久重定向
     - 302 Found：临时重定向
     - 304 Not Modified：使用缓存

   - **4xx 客户端错误：**
     - 400 Bad Request：请求参数错误
     - 401 Unauthorized：未授权
     - 403 Forbidden：禁止访问
     - 404 Not Found：资源不存在

   - **5xx 服务器错误：**
     - 500 Internal Server Error：服务器内部错误
     - 502 Bad Gateway：网关错误
     - 503 Service Unavailable：服务不可用

4. **HTTP缓存机制** → [协商缓存和跨域面试题.md](./协商缓存和跨域面试题.md)

   **强缓存：**
   - Cache-Control：max-age=3600、no-cache、no-store
   - Expires：过期时间（HTTP/1.0）
   - 命中强缓存：直接使用缓存，状态码200（from cache）

   **协商缓存：**
   - ETag / If-None-Match：文件内容哈希
   - Last-Modified / If-Modified-Since：文件修改时间
   - 命中协商缓存：返回304 Not Modified

   **缓存流程：**
   1. 浏览器发起请求
   2. 检查强缓存（Cache-Control、Expires）
   3. 未过期：直接使用缓存
   4. 已过期：发送协商缓存请求（带If-None-Match或If-Modified-Since）
   5. 服务器判断资源是否修改
   6. 未修改：返回304，使用缓存
   7. 已修改：返回200和新资源

5. **跨域问题** → [协商缓存和跨域面试题.md](./协商缓存和跨域面试题.md#三跨域问题)

   **同源策略：**
   - 协议、域名、端口三者完全相同

   **跨域解决方案：**

   - **CORS（跨域资源共享）**：
     ```javascript
     // 服务器设置响应头
     Access-Control-Allow-Origin: *
     Access-Control-Allow-Methods: GET, POST, PUT
     Access-Control-Allow-Headers: Content-Type
     Access-Control-Allow-Credentials: true
     ```

   - **JSONP**：
     - 原理：利用script标签不受同源策略限制
     - 缺点：只支持GET请求
     ```javascript
     function jsonp(url, callback) {
       const script = document.createElement('script');
       script.src = `${url}?callback=${callback}`;
       document.body.appendChild(script);
     }
     ```

   - **代理服务器**：
     - webpack-dev-server proxy
     - Vite proxy
     - nginx反向代理

6. **AJAX** → [AJAX经典面试题.md](./AJAX经典面试题.md)

   **XMLHttpRequest：**
   ```javascript
   const xhr = new XMLHttpRequest();
   xhr.open('GET', '/api/data');
   xhr.onreadystatechange = function() {
     if (xhr.readyState === 4 && xhr.status === 200) {
       console.log(xhr.responseText);
     }
   };
   xhr.send();
   ```

   **五种状态：**
   - 0 UNSENT：未初始化
   - 1 OPENED：已调用open
   - 2 HEADERS_RECEIVED：已接收响应头
   - 3 LOADING：正在接收响应体
   - 4 DONE：请求完成

   **Fetch API：**
   ```javascript
   fetch('/api/data')
     .then(response => response.json())
     .then(data => console.log(data))
     .catch(error => console.error(error));
   ```

### 第四模块：Vue进阶（1.5小时）

#### ✅ 必学知识点

1. **Vue生命周期** → [Vue面试总结大全.md](../面试总结大全/Vue面试总结大全.md#1-vue生命周期) + [Vue生命周期面试题大全.md](./Vue生命周期面试题大全.md)

   **Vue3生命周期钩子：**
   - setup：组合式API入口，最先执行
   - onBeforeMount：挂载前
   - onMounted：挂载后（可访问DOM）
   - onBeforeUpdate：更新前
   - onUpdated：更新后
   - onBeforeUnmount：卸载前
   - onUnmounted：卸载后

   **Vue2生命周期钩子：**
   - beforeCreate、created
   - beforeMount、mounted
   - beforeUpdate、updated
   - beforeDestroy、destroyed

   **父子组件生命周期执行顺序：**
   - 创建：父setup → 父onBeforeMount → 子setup → 子onBeforeMount → 子onMounted → 父onMounted
   - 更新：父onBeforeUpdate → 子onBeforeUpdate → 子onUpdated → 父onUpdated
   - 销毁：父onBeforeUnmount → 子onBeforeUnmount → 子onUnmounted → 父onUnmounted

2. **路由模式** → [Vue面试总结大全.md](../面试总结大全/Vue面试总结大全.md#五vue路由)

   **Hash模式：**
   - URL格式：`http://example.com/#/user`
   - 原理：监听hashchange事件
   - 优点：兼容性好、无需服务器配置
   - 缺点：URL不美观、SEO不友好

   **History模式：**
   - URL格式：`http://example.com/user`
   - 原理：HTML5 History API（pushState、replaceState）
   - 优点：URL美观、SEO友好
   - 缺点：需要服务器配置（404 fallback）

3. **路由守卫** → [Vue面试总结大全.md](../面试总结大全/Vue面试总结大全.md#3-路由守卫)

   **全局守卫：**
   ```javascript
   router.beforeEach((to, from, next) => {
     if (to.meta.requiresAuth && !isLoggedIn()) {
       next('/login');
     } else {
       next();
     }
   });

   router.afterEach((to, from) => {
     // 路由跳转后执行
   });
   ```

   **路由独享守卫：**
   ```javascript
   {
     path: '/admin',
     component: Admin,
     beforeEnter: (to, from, next) => {
       // 路由配置中定义
     }
   }
   ```

   **组件内守卫：**
   ```javascript
   export default {
     beforeRouteEnter(to, from, next) {},
     beforeRouteUpdate(to, from, next) {},
     beforeRouteLeave(to, from, next) {}
   }
   ```

4. **Vuex/Pinia状态管理** → [Vuex和Pinia面试题精简版.md](./Vuex和Pinia面试题精简版.md)

   **Vuex核心概念：**
   - State：状态数据
   - Getters：计算属性
   - Mutations：同步修改state
   - Actions：异步操作，提交mutation
   - Modules：模块化

   **Pinia优势：**
   - 更简洁的API，无需mutations
   - 完整的TypeScript支持
   - 自动代码分割
   - 更好的DevTools支持

   **Pinia基本使用：**
   ```javascript
   import { defineStore } from 'pinia';

   export const useUserStore = defineStore('user', {
     state: () => ({
       name: '',
       age: 0
     }),
     getters: {
       isAdult: (state) => state.age >= 18
     },
     actions: {
       updateName(name) {
         this.name = name;
       }
     }
   });
   ```
