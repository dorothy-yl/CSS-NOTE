## 📖 详细学习路径

### 第一部分：HTML进阶（0.5小时）

#### ✅ 必学知识点

1. **本地存储对比**
   - localStorage：
     - 持久化存储，不过期
     - 容量约5-10MB
     - 同源页面共享
     - 使用场景：用户配置、token
   - sessionStorage：
     - 会话存储，关闭标签页清除
     - 容量约5-10MB
     - 仅当前标签页可用
     - 使用场景：表单数据、临时状态
   - Cookie：
     - 容量小(4KB)
     - 每次请求自动携带
     - 可设置过期时间
     - 使用场景：身份认证

2. **HTML5实用API**
   - Geolocation API：获取用户地理位置
   - Drag and Drop API：拖放功能
   - Canvas API：绘制图形、图表、游戏
   - Web Workers：后台线程，处理耗时任务
   - Service Worker：离线缓存、消息推送
   - History API：pushState、replaceState操作历史记录

### 第二部分：CSS补充（1小时）

#### ✅ 必学知识点

1. **CSS3动画与过渡**
   - transition过渡：
     ```css
     .box {
       transition: all 0.3s ease-in-out;
     }
     .box:hover {
       transform: scale(1.2);
     }
     ```
   - animation动画：
     ```css
     @keyframes fadeIn {
       from { opacity: 0; }
       to { opacity: 1; }
     }
     .box {
       animation: fadeIn 1s ease-out;
     }
     ```
   - transform变换：translate、rotate、scale、skew
   - 性能优化：优先使用transform和opacity（触发GPU加速）

2. **移动端适配**
   - rem适配：
     ```javascript
     function setRem() {
       const baseSize = 16;
       const scale = document.documentElement.clientWidth / 375;
       document.documentElement.style.fontSize = baseSize * scale + 'px';
     }
     ```
   - vw/vh适配：1vw = 视口宽度的1%
   - 1px问题：transform: scaleY(0.5)
   - viewport设置：
     ```html
     <meta name="viewport" content="width=device-width, initial-scale=1.0">
     ```

3. **CSS预处理器**
   - Less/Sass变量：`$primary-color: #007bff;`
   - 嵌套：简化选择器书写
   - Mixin混入：复用样式片段
   - 函数：lighten()、darken()、透明度调整

### 第三部分：JavaScript补充（1.5小时）

#### ✅ 必学知识点

1. **深拷贝与浅拷贝** → [JS高频面试题汇总.md](./JS高频面试题汇总.md#10-深拷贝与浅拷贝-)
   - 浅拷贝：
     ```javascript
     const copy1 = { ...obj };
     const copy2 = [...arr];
     const copy3 = Object.assign({}, obj);
     ```
   - 深拷贝：
     ```javascript
     // 方法1：JSON（有局限）
     const deepCopy1 = JSON.parse(JSON.stringify(obj));

     // 方法2：递归实现
     function deepClone(obj, map = new WeakMap()) {
       if (obj === null || typeof obj !== 'object') return obj;
       if (map.has(obj)) return map.get(obj);

       const clone = Array.isArray(obj) ? [] : {};
       map.set(obj, clone);

       for (let key in obj) {
         if (obj.hasOwnProperty(key)) {
           clone[key] = deepClone(obj[key], map);
         }
       }
       return clone;
     }

     // 方法3：structuredClone（新API）
     const deepCopy2 = structuredClone(obj);
     ```

2. **数组常用方法**
   - 遍历方法：forEach、map、filter、reduce、some、every、find、findIndex
   - 修改原数组：push、pop、shift、unshift、splice、sort、reverse
   - 不修改原数组：concat、slice、join、map、filter
   - 实用技巧：
     ```javascript
     // 数组去重
     const unique = [...new Set(arr)];

     // 数组扁平化
     const flat = arr.flat(Infinity);

     // 数组求和
     const sum = arr.reduce((acc, cur) => acc + cur, 0);
     ```

3. **手写经典代码**
   - 手写防抖：
     ```javascript
     function debounce(fn, delay) {
       let timer = null;
       return function(...args) {
         if (timer) clearTimeout(timer);
         timer = setTimeout(() => {
           fn.apply(this, args);
         }, delay);
       };
     }
     ```
   - 手写节流：
     ```javascript
     function throttle(fn, delay) {
       let last = 0;
       return function(...args) {
         const now = Date.now();
         if (now - last >= delay) {
           fn.apply(this, args);
           last = now;
         }
       };
     }
     ```
   - 手写Promise.all：
     ```javascript
     Promise.myAll = function(promises) {
       return new Promise((resolve, reject) => {
         const results = [];
         let count = 0;
         for (let i = 0; i < promises.length; i++) {
           Promise.resolve(promises[i]).then(res => {
             results[i] = res;
             count++;
             if (count === promises.length) resolve(results);
           }).catch(reject);
         }
       });
     };
     ```

### 第四部分：Vue补充（1小时）

#### ✅ 必学知识点

1. **Vue Router深入** → [Vue完整面试题汇总.md](./Vue完整面试题汇总.md#16-路由传参)
   - 路由模式：
     - hash模式：URL带#，不需要服务器配置
     - history模式：干净URL，需要服务器配置重定向
   - 路由守卫：
     - 全局守卫：beforeEach、afterEach
     - 路由独享守卫：beforeEnter

   - 路由懒加载：
     ```javascript
     const Home = () => import('./views/Home.vue');
     ```

2. **keep-alive缓存机制**
   - 作用：缓存组件状态，避免重复渲染
   - 属性：include、exclude、max
   - 生命周期：activated、deactivated
   - 使用场景：列表页和详情页切换

3. **Diff算法原理** → [diff算法和虚拟DOM面试题汇总.md](./diff算法和虚拟DOM面试题汇总.md)
   - 虚拟DOM：用JavaScript对象描述DOM结构
   - Diff算法目的：找出新旧虚拟DOM的差异，最小化DOM操作
   - Vue2的双端比较：
     - 头头比较、尾尾比较
     - 头尾比较、尾头比较
     - 乱序比较（key的作用）
   - Vue3的最长递增子序列：
     - 减少节点移动次数
     - 性能更优
   - key的作用：
     - 唯一标识节点
     - 复用和重新排序现有元素
     - 不要用index作为key（列表会变化时）

### 第五部分：HTTP网络（2小时）

#### ✅ 必学知识点

1. **HTTP vs HTTPS** → [HTTP面试题精简版.md](./HTTP面试题精简版.md#1-http-vs-https-)
   - 端口区别：HTTP 80、HTTPS 443
   - 安全性：HTTPS有SSL/TLS加密，HTTP明文传输
   - 加密方式：对称加密（数据传输）+ 非对称加密（密钥交换）
   - SSL/TLS握手过程：客户端hello → 服务器证书 → 密钥交换 → 加密通信
   - CA证书作用：验证服务器身份，防止中间人攻击

2. **GET vs POST** → [HTTP面试题精简版.md](./HTTP面试题精简版.md#2-get-vs-post-)
   - 参数位置：GET在URL、POST在请求体
   - 数据大小：GET受URL长度限制、POST理论无限制
   - 幂等性：GET幂等、POST非幂等
   - 缓存：GET可缓存、POST默认不缓存
   - RESTful API：GET获取、POST创建、PUT更新、DELETE删除

3. **HTTP状态码** → [HTTP面试题精简版.md](./HTTP面试题精简版.md#4-http-状态码-)
   - 2xx 成功：200、201、204
   - 3xx 重定向：301、302、304
   - 4xx 客户端错误：400、401、403、404
   - 5xx 服务器错误：500、502、503

4. **HTTP缓存机制** → [协商缓存和跨域面试题.md](./协商缓存和跨域面试题.md#一强缓存)
   - 强缓存：Cache-Control、Expires
   - 协商缓存：ETag/If-None-Match、Last-Modified/If-Modified-Since
   - 缓存流程：
     1. 先检查强缓存，未过期直接使用
     2. 强缓存过期，发送协商缓存请求
     3. 服务器返回304使用缓存，或200返回新内容

5. **跨域问题** → [协商缓存和跨域面试题.md](./协商缓存和跨域面试题.md#三跨域问题)
   - 同源策略：协议、域名、端口三者完全相同
   - CORS：Access-Control-Allow-Origin等响应头
   - JSONP：利用script标签不受同源策略限制，只支持GET
   - 代理服务器：webpack-dev-server、Vite的proxy、nginx反向代理

6. **AJAX和Fetch API** → [AJAX经典面试题.md](./AJAX经典面试题.md)
   - XMLHttpRequest的五种状态：0-4
   - Fetch API优势：基于Promise、更简洁
   - Fetch注意事项：默认不发送Cookie、HTTP错误不reject

### 第六部分：性能优化（2小时）

#### ✅ 必学知识点

1. **长列表优化** → [前端性能优化面试题精简版.md](./前端性能优化面试题精简版.md#三javascript优化)
   - 虚拟滚动（Virtual Scroll）：
     - 原理：只渲染可视区域的列表项
     - 计算：可视区域高度 / 每项高度 = 可见数量
     - 滚动时动态计算startIndex和endIndex
   - 时间分片（Time Slicing）：
     ```javascript
     function timeSlice(list, fn, count = 100) {
       let index = 0;
       function chunk() {
         for (let i = 0; i < count && index < list.length; i++, index++) {
           fn(list[index]);
         }
         if (index < list.length) {
           requestAnimationFrame(chunk);
         }
       }
       chunk();
     }
     ```
   - 分页加载和无限滚动

2. **Webpack打包优化** → [前端性能优化面试题精简版.md](./前端性能优化面试题精简版.md#七webpackvite优化)
   - 代码分割（Code Splitting）：splitChunks配置
   - Tree Shaking：删除未使用的代码，需要ES Module
   - 缓存优化：contenthash、持久化缓存
   - 构建速度优化：thread-loader、cache-loader、缩小范围

3. **Vite构建优化**
   - Vite vs Webpack：
     - 开发环境：Vite用ES Module，按需编译，速度极快
     - 生产环境：Vite用Rollup，Webpack用自身打包
   - 依赖预构建：使用esbuild（Go语言）预构建依赖
   - 热更新（HMR）：只更新修改的模块，不刷新页面

4. **前端监控** → [前端性能优化面试题精简版.md](./前端性能优化面试题精简版.md#一性能指标与衡量标准)
   - 性能指标：FP、FCP、LCP、CLS、TTI
   - 错误监控：window.onerror、Promise错误
   - 性能监控：window.performance、PerformanceObserver
   - 常见平台：Sentry、Fundebug
